# Doc-Memory-Generation

基于上下文信标压缩的记忆-生成解耦长文档检索增强框架。

## 问题与动机

长文档检索增强生成（Long-Document RAG）面临上下文编码与知识解码协同不足的问题：
- 传统方案将长文档全文编码，KV缓存开销随文档长度线性增长
- 检索侧与记忆侧独立运作，缺乏多层次的知识融合机制
- 通用模型在专业知识密集领域的知识召回能力有限

## 方法

针对上述问题，提出**上下文信标压缩 + 记忆-生成解耦**的长文档检索增强框架：

### 整体架构

```
长文档 → [信标压缩编码] → 压缩记忆向量
                              ↓
用户查询 → [多粒度记忆探针] → 检索线索 → [稠密检索] → 混合知识 → [生成解码] → 答案
```

### 核心模块

1. **上下文信标压缩**：在输入序列中插入信标token，通过分段注意力汇聚将长文档映射为紧凑记忆向量

2. **多粒度记忆查询**：从压缩记忆中以三类探针并行抽取检索线索——文本片段召回、替身查询改写、关键点摘要

3. **记忆召回-稠密检索混合知识构建**：将记忆侧导出的结构化线索与稠密检索证据融合

4. **记忆-生成参数分离训练**：冻结生成模块通用参数，仅微调记忆编码参数

## 实验结果

在非遗专业领域4类问答任务上与通用长文档基线对比：

| 任务 | Baseline | Ours | 提升 |
|------|----------|------|------|
| 流程推演 | 1.10 | 1.57 | +42.7% |
| 派别对比 | 3.07 | 3.04 | -1.0% |
| 术语辨义 | 2.61 | 2.69 | +3.1% |
| 知识溯源 | 2.33 | 2.99 | +28.3% |
| **平均F1** | **2.28** | **2.57** | **+12.7%** |

## 关键发现

- 短文本场景下信标压缩模块因未达激活阈值导致记忆向量退化，动态分组拼接策略使训练样本全部覆盖激活区间
- 参数分离训练（冻结4.1B生成参数，仅微调462M记忆参数）相比全参数微调具有更好的知识保持能力

## 项目结构

```
├── src/                    # 核心代码
│   ├── memorag.py          # 记忆-生成双模型管线
│   ├── prompt.py           # 多粒度记忆探针模板
│   ├── retrieval.py        # 稠密检索模块
│   └── agent.py            # 智能体调度
├── scripts/                # 评估与数据处理
│   ├── eval_ich.py         # 非遗领域评估脚本
│   └── prepare_ich_data.py # 数据准备管线
├── configs/                # 训练配置
│   └── ich-pretrain-qwen2.sh
├── data/                   # 数据
│   ├── train.json          # 训练语料
│   └── ich_qa.json         # 专业领域问答测试集
└── results/                # 实验结果
```

## 依赖

基于 [MemoRAG](https://github.com/qhjqhj00/MemoRAG) 框架扩展，核心依赖：

- PyTorch >= 2.0
- Transformers >= 4.36
- FAISS-CPU
- BGE-M3 (检索模型)
- Mistral-7B / Qwen2-7B
