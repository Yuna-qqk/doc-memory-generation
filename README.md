# 基于上下文信标压缩的记忆-生成解耦长文档检索增强方法研究

## 项目概述

针对长文档检索增强生成中上下文编码与知识解码协同不足的问题，提出基于上下文信标压缩的记忆-生成解耦框架。在4个非遗专业领域问答任务上验证有效性，平均F1提升12.7%，其中流程推演与知识溯源分别提升42.7%和28.3%。

## 方法架构

```
长文档 → [信标压缩编码] → 压缩记忆向量
                              ↓
用户查询 → [多粒度记忆探针] → 检索线索 → [稠密检索] → 混合知识 → [生成解码] → 答案
```

### 1. 多粒度记忆查询算法

设计文本片段召回、替身查询改写与关键点摘要三类记忆探针，从压缩记忆中并行抽取不同粒度的检索线索，替代传统单一原文检索策略。

### 2. 记忆召回-稠密检索混合知识构建

将压缩记忆导出的片段线索、改写查询与稠密检索结果融合为结构化知识上下文，替代传统纯检索知识拼接。揭示短文本输入下信标压缩因未达激活阈值导致记忆向量退化的失效机制，设计动态分组拼接算法使全部训练样本覆盖信标激活区间。

### 3. 记忆-生成参数分离训练

冻结4.1B生成模块参数，仅微调462M信标压缩参数，在低资源约束下验证解耦训练相比全参数微调的知识保持优势与训练稳定性。

## 实验结果

在非遗专业领域4类问答任务上与通用长文档基线对比：

| 指标 | 数值 |
|------|------|
| 平均F1 | 2.57 (+12.7%) |
| 流程推演 | +42.7% |
| 知识溯源 | +28.3% |
| 损失收敛 | 5.67 → 2.31 |

## 项目结构

```
├── src/                    # 核心算法
│   ├── memorag.py          # 记忆-生成双模型管线
│   ├── prompt.py           # 多粒度记忆探针模板
│   ├── retrieval.py        # 稠密检索模块
│   └── agent.py            # 智能体调度
├── scripts/                # 评估与数据处理
│   ├── eval_ich.py         # 领域评估脚本
│   └── prepare_ich_data.py # 数据准备管线
├── configs/                # 训练配置
├── data/                   # 数据
│   ├── train.json          # 领域训练语料
```
